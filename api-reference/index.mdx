MVSoftware exposes 75+ API endpoints through Next.js API routes. All endpoints are organized under `/api/`.

## Authentication patterns

| Pattern | Used by | Mechanism |
|---------|---------|-----------|
| **Session auth** | Most endpoints | NextAuth JWT session via `getServerSession()` |
| **Cron auth** | Scheduled tasks | `X-Cron-Secret` header or Bearer token matching `CRON_SECRET` |
| **Webhook signature** | Jobber webhooks | HMAC-SHA256 signature verification |
| **Public** | Health check | No auth required |

## Role-based access

Session-authenticated endpoints enforce role requirements:

| Role | Access |
|------|--------|
| `admin` | Full access including settings, export, billing data |
| `approver` | Reports, document review, classification overrides |
| `viewer` | Limited read-only access to own data |

## Common patterns

### Error responses

All endpoints use standard HTTP status codes:

| Status | Meaning |
|--------|---------|
| 400 | Bad request (missing or invalid parameters) |
| 401 | Not authenticated |
| 403 | Insufficient permissions |
| 404 | Resource not found |
| 422 | Validation error |
| 429 | Rate limited |
| 500 | Internal server error |

### Streaming responses

Some endpoints use NDJSON (newline-delimited JSON) for streaming:
- `/api/timetrack/sync` — progress events during ClickUp sync
- `/api/timetrack/classify/stream` — streaming classification results

Each line is a complete JSON object.

### Pagination

Paginated endpoints accept:
- `limit` — items per page (1-100, default 50)
- `offset` — skip N items (default 0)

### Batch operations

Batch endpoints accept arrays with size caps (typically 500 items max) to prevent request timeouts.
