MVSoftware uses Supabase (PostgreSQL) as its central data store. Schema changes are managed through migration files in `supabase/migrations/`.

## Vantaca-Jobber tables

<Accordion title="vantaca_work_orders">
  Stores work orders synced from the Vantaca API. Each row represents a single work order with its current sync state.

  Key columns: `xn_number` (unique identifier), `step_id`, `description`, `community_name`, `sync_status`, `last_synced_at`
</Accordion>

<Accordion title="jobber_jobs">
  Maps Vantaca work orders to Jobber jobs. Created when a work order enters a syncable step.

  Key columns: `jobber_id`, `work_order_id` (FK), `status`, `created_at`
</Accordion>

<Accordion title="jobber_job_mappings">
  Cross-reference lookup between Vantaca XN numbers and Jobber job IDs for fast bidirectional queries.
</Accordion>

<Accordion title="jobber_users">
  Cached Jobber team member data for assignment and display without repeated API calls.
</Accordion>

<Accordion title="jobber_oauth_tokens">
  OAuth token storage with refresh logic. Stores access tokens, refresh tokens, and expiration timestamps.
</Accordion>

<Accordion title="work_order_attachments">
  File metadata for attachments synced from Vantaca, stored in Supabase Storage, and linked to Jobber jobs.
</Accordion>

## TimeTrack tables

<Accordion title="time_entries">
  Individual time entries synced from ClickUp. The largest table in the system.

  Key columns: `clickup_id`, `user_id`, `description`, `duration_ms`, `billable`, `ai_classification`, `ai_confidence`, `client_id`, `task_type`, `approved`, `exported`
</Accordion>

<Accordion title="entry_flags">
  Quality flags attached to time entries. Entries with blocking flags cannot be exported.

  Key columns: `entry_id` (FK), `flag_type`, `blocking`, `resolved`, `resolved_by`, `resolution_note`

  See [flag types](/timetrack/flags) for all 16 flag types and their blocking behavior.
</Accordion>

<Accordion title="entry_notes">
  Clarifications, corrections, and reviewer notes attached to time entries.

  Key columns: `entry_id` (FK), `author_id`, `note_type`, `content`, `created_at`
</Accordion>

<Accordion title="employee_rates">
  Per-client billing rates with override support. Used by the billability engine.

  Key columns: `employee_id`, `client_id`, `hourly_rate`, `effective_date`, `override_reason`
</Accordion>

<Accordion title="client_quotas">
  Budget and time limits per client for cost tracking and alerts.
</Accordion>

<Accordion title="capital_project_entries">
  Links time entries to fixed assets for depreciation tracking.

  Key columns: `entry_id` (FK), `project_id`, `asset_category`, `cost_allocation`
</Accordion>

<Accordion title="gl_codes">
  General ledger code mappings for QuickBooks export.
</Accordion>

<Accordion title="approval_workflows">
  Approval queue state for time entries requiring review.

  Key columns: `entry_id` (FK), `status`, `assigned_to`, `escalated_at`, `completed_at`
</Accordion>

## Scrubbing tables

<Accordion title="scrubbing_jobs">
  Kanban job records with 11 status columns tracking jobs from field completion to invoicing.

  Key columns: `jobber_id`, `status`, `assigned_to`, `client_name`, `claimed_at`, `submitted_at`
</Accordion>

<Accordion title="scrubbing_comments">
  Review notes, status change records, and resolutions attached to scrubbing jobs.

  Key columns: `job_id` (FK), `comment_type` (`note`, `review`, `status_change`, `system`), `author_id`, `content`
</Accordion>

<Accordion title="scrubbing_client_config">
  Client-specific approval routing and workflow configuration.
</Accordion>

## Receipt analysis tables

<Accordion title="receipts">
  Uploaded documents with extraction status tracking.

  Key columns: `file_path`, `vendor`, `total_amount`, `extraction_status`, `confidence_score`, `processed_at`
</Accordion>

<Accordion title="receipt_items">
  Extracted line items with AI confidence scores.

  Key columns: `receipt_id` (FK), `description`, `quantity`, `unit_price`, `upc`, `confidence`, `matched_canonical_id`
</Accordion>

<Accordion title="vendor_prices">
  Historical price tracking across vendors for comparison and bulk optimization.
</Accordion>

<Accordion title="stocking_recommendations">
  Van inventory suggestions generated by the stocking engine based on usage patterns.
</Accordion>

## Teams bot tables

<Accordion title="teams_bot_service_configuration">
  Webhook URLs and authentication config for the Teams bot.
</Accordion>

<Accordion title="teams_notification_queue">
  Pending notifications waiting for delivery. Processed by the `teams-notifications` cron job.
</Accordion>

## Migration management

Migrations live in `supabase/migrations/` and are applied through the Supabase CLI:

```bash
# Create a new migration
npx supabase migration new <migration-name>

# Apply pending migrations
npx supabase db push

# Reset and reapply all migrations (development only)
npx supabase db reset
```

<Warning>
  Never run `db reset` against a production database. Always apply migrations forward with `db push`.
</Warning>
